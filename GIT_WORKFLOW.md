# 🚀 초보자를 위한 Git 워크플로우 가이드

이 문서는 `develop` 브랜치에서 안전하게 작업하고, 완성된 내용을 `main` 브랜치에 반영하는 핵심 단계를 설명합니다.

---

## 1. 현재 브랜치 확인하기

작업을 시작하기 전에 내가 어떤 브랜치에 있는지 꼭 확인하세요.

```bash
git branch
```
*결과에 `* develop`와 같이 표시되어야 합니다.*

---

## 2. develop에서 작업하기

`develop` 브랜치에서 코드를 수정하거나 새로운 기능을 추가합니다.

### 코드 변경 사항 저장하기 (Commit)
작업이 어느 정도 완료되면 변경 사항을 기록합니다.

1. **변경 내용 스테이징**: `git add .` (모든 변경 사항 선택)
2. **기록 남기기**: `git commit -m "기능 추가: 로그인 화면 구현"`

---

## 3. main 브랜치로 합치기 (Merge)

`develop`에서의 작업이 모두 끝나고 검토가 완료되었다면, 이제 `main` 브랜치에 반영할 차례입니다.

### 📋 단계별 실행

1. **main 브랜치로 이동**:
   ```bash
   git checkout main
   ```

2. **최신 상태 업데이트 (권장)**:
   (원격 저장소가 있는 경우)
   ```bash
   git pull origin main
   ```

3. **develop의 내용을 main으로 병합**:
   ```bash
   git merge develop
   ```

4. **최종 결과 업로드**:
   (원격 저장소에 반영할 때)
   ```bash
   git push origin main
   ```


---

## 4. Firebase Hosting에 배포하기 (Deploy)

`main` 브랜치에 코드를 합쳤다면, 이제 실제 웹사이트(호스팅)에 반영할 차례입니다.

### 📋 배포 단계

1. **최신 코드 빌드**:
   Next.js 프로젝트를 정적 파일로 변환합니다.
   ```bash
   npm run build
   ```
   *성공하면 프로젝트 폴더에 `out` 폴더가 
   확인: 명령어 실행 결과로 나온 Hosting URL에 접속하여 변경 사항이 반영되었는지 확인하세요.
   
   생성되거나 업데이트됩니다.*

2. **Firebase 배포**:
   빌드된 파일을 호스팅 서버로 업로드합니다.
   ```bash
   firebase deploy --only hosting
   ```

3. **확인**:
   명령어 실행 결과로 나온 `Hosting URL`에 접속하여 변경 사항이 반영되었는지 확인하세요.
