# 🚀 초보자를 위한 Git 워크플로우 가이드

이 문서는 `develop` 브랜치에서 안전하게 작업하고, 완성된 내용을 `main` 브랜치에 반영하는 핵심 단계를 설명합니다.

---

## 1. 현재 브랜치 확인하기

작업을 시작하기 전에 내가 어떤 브랜치에 있는지 꼭 확인하세요.

```bash
git branch
```
*결과에 `* develop`와 같이 표시되어야 합니다.*

---

## 2. develop에서 작업하기

`develop` 브랜치에서 코드를 수정하거나 새로운 기능을 추가합니다.

### 코드 변경 사항 저장하기 (Commit)
작업이 어느 정도 완료되면 변경 사항을 기록합니다.

1. **변경 내용 스테이징**: `git add .` (모든 변경 사항 선택)
2. **기록 남기기**: `git commit -m "기능 추가: 로그인 화면 구현"`

---

## 3. main 브랜치로 합치기 (Merge)

`develop`에서의 작업이 모두 끝나고 검토가 완료되었다면, 이제 `main` 브랜치에 반영할 차례입니다.

### 📋 단계별 실행

1. **main 브랜치로 이동**:
   ```bash
   git checkout main
   ```

2. **최신 상태 업데이트 (권장)**:
   (원격 저장소가 있는 경우)
   ```bash
   git pull origin main
   ```

3. **develop의 내용을 main으로 병합**:
   ```bash
   git merge develop
   ```

4. **최종 결과 업로드**:
   (원격 저장소에 반영할 때)
   ```bash
   git push origin main
   ```

---

## 💡 유의사항

- **충돌(Conflict)**: 만약 `main`과 `develop`에서 같은 파일의 같은 부분을 수정했다면 충돌이 발생할 수 있습니다. 이 경우 에디터에서 코드를 정리한 후 다시 `add`와 `commit`을 해주세요.
- **다시 작업하러 가기**: `main`에 성공적으로 합쳤다면 다시 `develop`으로 돌아가 다음 작업을 이어가세요.
  ```bash
  git checkout develop
  ```
